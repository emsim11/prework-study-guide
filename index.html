HTML

Overview

  • Hypertext Markup Language is one of three core languages used in web development: HTML, CSS, and JavaScript.
  • HTML is used to create the structure of a website.
  • HTML elements provide content and the basic framework for all pages on the website.
  • The first line of code in the file is the !DOCTYPE, which tells the browser what documet type it should expect when it is opened.
  • For HTML, the !DOCTYPE html declaration tells the browser to expect HTML on the page.
  • The next line of code has the HTML opening tag, which represents the root, or start, of the HTML document.
  • Inside this HTML opening tag, we designate an attribute of lang="en", which tells the browser that the language for the webpage is English.
  • HTML attributes define additional information about an element, and they are always defined in the BEGINNING ELEMENT.

Head Element

  • The head element contains information, or metadata, for the web application.
  • Metadata is data about our webpage used by the browser to effectively serve the webpage to users.
  • Typically, metadata is not seen by users, but it contains important information about accessibility, search engines, and performance.
  • Inside the head element, we can use the following metadata elements: title, style, base, link, meta, and script.
  • The title element defines the title of the document, showing it in the webpage's tab. This element is required and is important for Search Engine Optimization.
  • The style element defines the CSS styles associated with a website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata, such as the character set, description, keywords, author, and viewport.
  • Meta element examples: The charset element sets the character encoding; The http-equiv element sets an HTTP header for content; The viewport element sets the default view for the webpage.
  • The script element embeds client-side scripts in the HTML document.
  • All of these meta elements contain information about the page that is used by the browser, but won't be visible to the user.

Body Element

  • The body element of an HTML document contains the content shown to users when they look at the webpage in a browser.
  • All elements that we want the user to see and interact with must be placed inside the body.
  • Elements that are nested inside other elements are called child elements; the element that holds all the child elements is known as the parent element.
  • The body element defines the body of the document, and it is an example of a semantic HTML element.
  • Semantic HTML elements are used to indicate the role of their content, as well as to give structure to the content of the webpage.
  • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage.
  • Using semantic HTML elements in your code makes your HTML easier to read by other developers and the browser. It also plays an important role in assuring that your webpage is accessible to the widest variety of users, including those who use assistive technologies such as screen readers.
  • Examples of non-semantic elements include div, which creates a division in the page, and span, which creates an inline division.
  • It is best practice to use semantic elements whenever possible - especially when the content is visible to users.
  • Content within visible HTML elements is contained between an opening tag and a closing tag. These tags are often represented by angle brackets with associated tag names contained inside. Closing tags begin with a forward slash. Content is inserted between an opening tag and a closing tag so that the browser understands how to treat the content.
  • Note: An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what appears inside the angle brackets.
  • H1, H2, H3, H4, H5, and H6 elements represent the level of heading a given text block represents, with H1 being the largest. There should only be one H1 in the HTML document.
  • The p element represents a paragraph or a block of text, and it is used extensively to wrap most of the text on the webpages.
  • The ul, ol, and li HTML elements represent lists. The ul element renders an unordered, or bullet, list. The ol element renders a numbered, or numerical, list. The li TAG is used inside ordered or unordered lists to define a list item.
  • The img element contains information about images that are displayed on the webpage.
  • The img element has the following attributes: src and alt.
  • The src attribute is the most important, as it defines the location, or source, of the image file.
  • The alt attribute contains a text string that describes the appearance and functionality of the image. The alt text is important for screen readers and other assistive technologies. It can also improve search engine rankings for your website.
  • The br element creates a line of empty space, or a line break, between two blocks of content.
  • The a element, which stands for anchor, creates links to the same webpage or other webpages.
  • The href attribute points to the URL for the link.
  • Section elements are semantic elements that define a section in a document that contains thematic content. They elegantly organize the content on the page based on the relevant topics.
  • Each section element has an id attribute that matches the topic. HTML uses the id attribute to identify elements and select them.
  • Each section element also has a class attribute, which works in a similar way to the id.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout branch-name: switches to the branch
  • git clone ssh: clones a git repository using SSH
  • git add file-name file-name: stages the file/files and their changes to the current, working branch
  • git add -A: stages all files and their changes in the working branch
  • git commit -m: commits changes that have been staged to the repository; use the -m flag to associate a message with the commit that is a short descriptive message of the changes implemented
  • git pull origin main: used to check if current, local branch is up to date/synced with the remote branch in GitHub before opening a pull request; this is extremely important in a team environment
  • ***It is a good habit to always pull the base branch into your feature branch before pushing up any code. You should see a message in your terminal after git pull origin main that says "Already up to date."***
  • git push origin feature/feature-name: pushes up all of the changes to the remote repository
